## Get an Online Table

Public preview

`GET/api/2.0/online-tables/{name}`

Get information about an existing online table and its status.

API scopes (preview):[`unity-catalog`](https://docs.databricks.com/api/workspace/api/scopes#unity-catalog)

### Path parameters

[`name`](https://docs.databricks.com/api/workspace/onlinetables/get#name)required string

Full three-part (catalog, schema, table) name of the table.

### Responses

**200** Request completed successfully.

Request completed successfully.

[`name`](https://docs.databricks.com/api/workspace/onlinetables/get#name)string

Full three-part (catalog, schema, table) name of the table.

[`spec`](https://docs.databricks.com/api/workspace/onlinetables/get#spec)object

Specification of the online table.

[`perform_full_copy`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-perform_full_copy)boolean

Default`false`

Whether to create a full-copy pipeline -- a pipeline that stops after creates a full copy of the source table upon initialization and does not process any change data feeds (CDFs) afterwards. The pipeline can still be manually triggered afterwards, but it always perform a full copy of the source table and there are no incremental updates. This mode is useful for syncing views or tables without CDFs to online tables. Note that the full-copy pipeline only supports "triggered" scheduling policy.

[`pipeline_id`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-pipeline_id)string

ID of the associated pipeline. Generated by the server - cannot be set by the caller.

[`primary_key_columns`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-primary_key_columns)Array of string

Primary Key columns to be used for data insert/update in the destination.

[`run_continuously`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-run_continuously)object

Pipeline runs continuously after generating the initial data.

[`run_triggered`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-run_triggered)object

Pipeline stops after generating the initial data and can be triggered later (manually, through a cron job or through data triggers)

[`source_table_full_name`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-source_table_full_name)string

Three-part (catalog, schema, table) name of the source Delta table.

[`timeseries_key`](https://docs.databricks.com/api/workspace/onlinetables/get#spec-timeseries_key)string

Time series key to deduplicate (tie-break) rows with the same primary key.

[`status`](https://docs.databricks.com/api/workspace/onlinetables/get#status)object

Online Table data synchronization status

[`continuous_update_status`](https://docs.databricks.com/api/workspace/onlinetables/get#status-continuous_update_status)object

Detailed status of an online table. Shown if the online table is in the ONLINE_CONTINUOUS_UPDATE or the ONLINE_UPDATING_PIPELINE_RESOURCES state.

[`detailed_state`](https://docs.databricks.com/api/workspace/onlinetables/get#status-detailed_state)string

Enum: `PROVISIONING | PROVISIONING_PIPELINE_RESOURCES | PROVISIONING_INITIAL_SNAPSHOT | ONLINE | ONLINE_CONTINUOUS_UPDATE | ONLINE_TRIGGERED_UPDATE | ONLINE_NO_PENDING_UPDATE | OFFLINE | OFFLINE_FAILED | ONLINE_PIPELINE_FAILED | ONLINE_UPDATING_PIPELINE_RESOURCES`

The state of the online table.

[`failed_status`](https://docs.databricks.com/api/workspace/onlinetables/get#status-failed_status)object

Detailed status of an online table. Shown if the online table is in the OFFLINE_FAILED or the ONLINE_PIPELINE_FAILED state.

[`message`](https://docs.databricks.com/api/workspace/onlinetables/get#status-message)string

A text description of the current state of the online table.

[`provisioning_status`](https://docs.databricks.com/api/workspace/onlinetables/get#status-provisioning_status)object

Detailed status of an online table. Shown if the online table is in the PROVISIONING_PIPELINE_RESOURCES or the PROVISIONING_INITIAL_SNAPSHOT state.

[`triggered_update_status`](https://docs.databricks.com/api/workspace/onlinetables/get#status-triggered_update_status)object

Detailed status of an online table. Shown if the online table is in the ONLINE_TRIGGERED_UPDATE or the ONLINE_NO_PENDING_UPDATE state.

[`table_serving_url`](https://docs.databricks.com/api/workspace/onlinetables/get#table_serving_url)string

Data serving REST API URL for this table

[`unity_catalog_provisioning_state`](https://docs.databricks.com/api/workspace/onlinetables/get#unity_catalog_provisioning_state)string

Enum: `PROVISIONING | ACTIVE | FAILED | DELETING | UPDATING | DEGRADED`

The provisioning state of the online table entity in Unity Catalog. This is distinct from the state of the data synchronization pipeline (i.e. the table may be in "ACTIVE" but the pipeline may be in "PROVISIONING" as it runs asynchronously).
